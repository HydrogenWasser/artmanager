# 美术资源管理工具开发文档

版本：v0.1  
目标读者：Claude Code / 开发者  
目标平台：Windows 10 / Windows 11  
开发目标：实现一个轻量级、绑定真实文件系统的美术资源管理工具，用于管理游戏项目中的 Aseprite 文件、PNG 文件、AI 原图、预览图、动画序列帧、场景图等美术素材。

---

# 1. 项目背景

用户正在制作一个游戏，当前美术资源主要来源于以下流程：

1. 使用 ChatGPT 或其他 AI 图像工具生成原始像素图像。
2. 使用 Aseprite 对原始像素图像进行修改、绘制、切帧、导出。
3. 使用 Godot 作为游戏引擎，最终主要使用 PNG、spritesheet、序列帧等导出资源。
4. 当前所有美术资源直接通过 Windows 文件资源管理器管理，随着角色、场景、道具、动画越来越多，文件夹管理方式效率很低。

因此需要开发一个专门面向像素游戏开发的美术资源管理工具。

该工具不是网盘，不是素材导入库，也不是替代 Windows 文件系统的独立资源空间，而是一个绑定真实项目文件夹的可视化资源管理器。

---

# 2. 核心定位

本工具应被设计为：

> 一个绑定真实文件系统的轻量级美术资产浏览器和管理器。

核心原则：

1. 所有素材文件仍然保存在真实文件夹中。
2. 工具内部不复制一份独立素材库。
3. 工具只维护逻辑树、标签、备注、资源关系、缩略图缓存等辅助信息。
4. 工具中的添加、删除、重命名、移动文件操作，必须尽量等价于直接在 Windows 文件资源管理器中操作。
5. 工具应支持快速预览、快速搜索、快速打开 Aseprite、快速打开所在文件夹。
6. 工具要轻量，不能做成重型 DAM 系统。

第一版目标是做出一个真正可用的 MVP，而不是完整商业软件。

6. 支持向资源节点添加文件。
7. 支持删除文件到回收站。
8. 支持重命名文件。
9. 支持打开文件。
10. 支持用 Aseprite 打开 `.aseprite` / `.ase` 文件。
11. 支持打开文件所在文件夹，并选中文件。

# 7. 项目根目录设计

当用户选择一个美术资源根目录后，工具在该目录下创建 `.artmgr` 文件夹，用于存放工具自身数据。

示例：

```text
GameProject/
  Art/
    Characters/
    Scenes/
    Props/
    UI/

    .artmgr/
      artdb.sqlite
      config.json
      thumbnails/
        xxx.png
        yyy.png
```

注意：

1. `.artmgr` 只是工具辅助数据目录。
2. 不能把用户素材导入 `.artmgr`。
3. 用户素材必须继续保留在真实项目文件夹中。
4. 数据库中保存文件路径时，优先保存相对于项目根目录的相对路径。

# 8. 数据库设计

使用 SQLite。

数据库路径：

```text
<ArtRoot>/.artmgr/artdb.sqlite
```

## 8.1 asset_nodes 表

用于记录逻辑资源树。

```sql
CREATE TABLE IF NOT EXISTS asset_nodes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    parent_id INTEGER,
    name TEXT NOT NULL,
    type TEXT DEFAULT '',
    folder_path TEXT DEFAULT '',
    sort_order INTEGER DEFAULT 0,
    note TEXT DEFAULT '',
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL,
    FOREIGN KEY(parent_id) REFERENCES asset_nodes(id) ON DELETE CASCADE
);
```

字段说明：

```text
id：节点 ID
parent_id：父节点 ID
name：节点名称
type：节点类型（已废弃，保留字段用于兼容）
folder_path：自动计算的真实文件夹路径，相对项目根目录
sort_order：排序
note：备注
created_at：创建时间
updated_at：更新时间
```

> **说明**：节点类型系统已在后续版本中移除。所有节点不再区分类型，统一由层级结构管理。创建节点时 folder_path 由程序根据父节点路径自动推导。

## 8.2 asset_files 表

用于记录资源节点下关联的文件。

```sql
CREATE TABLE IF NOT EXISTS asset_files (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    node_id INTEGER NOT NULL,
    file_path TEXT NOT NULL,
    file_type TEXT NOT NULL,
    role TEXT DEFAULT '',
    thumbnail_path TEXT DEFAULT '',
    file_hash TEXT DEFAULT '',
    width INTEGER DEFAULT 0,
    height INTEGER DEFAULT 0,
    frame_count INTEGER DEFAULT 0,
    file_size INTEGER DEFAULT 0,
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL,
    FOREIGN KEY(node_id) REFERENCES asset_nodes(id) ON DELETE CASCADE
);
```

字段说明：

```text
node_id：所属资源节点
file_path：文件路径，相对项目根目录
file_type：文件类型，例如 image、aseprite、spritesheet、json、folder、other
role：文件角色
thumbnail_path：缩略图路径，相对 .artmgr/thumbnails
file_hash：文件 hash，用于判断缩略图是否过期
width：图片宽度
height：图片高度
frame_count：帧数，第一版可不严格实现
file_size：文件大小
created_at：记录创建时间
updated_at：记录更新时间
```

允许的 file_type：

```text
image
aseprite
spritesheet
json
gif
folder
other
```

允许的 role：

```text
ai_source
aseprite_source
export_png
preview
spritesheet
reference
mask
collision_ref
animation_frame
other
```

## 8.3 tags 表

```sql
CREATE TABLE IF NOT EXISTS tags (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL UNIQUE,
    color TEXT DEFAULT '#888888',
    created_at TEXT NOT NULL
);
```

## 8.4 asset_node_tags 表

```sql
CREATE TABLE IF NOT EXISTS asset_node_tags (
    node_id INTEGER NOT NULL,
    tag_id INTEGER NOT NULL,
    PRIMARY KEY(node_id, tag_id),
    FOREIGN KEY(node_id) REFERENCES asset_nodes(id) ON DELETE CASCADE,
    FOREIGN KEY(tag_id) REFERENCES tags(id) ON DELETE CASCADE
);
```

## 8.5 settings 表

```sql
CREATE TABLE IF NOT EXISTS settings (
    key TEXT PRIMARY KEY,
    value TEXT NOT NULL
);
```

用于保存：

```text
aseprite_path
thumbnail_size
default_open_mode
theme
last_opened_node
```

# 17. 交互细节

## 17.1 删除节点

删除节点时提供复选框选项：

```text
[ ] 同时删除磁盘上的对应文件夹
```

- 未勾选：仅删除数据库中的节点记录（级联删除子节点），磁盘文件保留。
- 勾选：先将该节点对应的磁盘文件夹发送到回收站，再删除数据库记录。

如果节点下有关联文件，子节点也会被级联删除。

## 17.2 删除文件

删除文件时提供两个选项：

```text
从当前节点移除引用
删除到回收站
```

默认建议使用“删除到回收站”，不要永久删除。

## 17.3 外部文件变化

如果用户在 Windows 资源管理器中删除了文件，工具应在下次刷新时发现文件缺失。

缺失文件卡片显示：

```text
文件缺失
```

并提供：

```text
移除记录
重新定位文件
```

第一版可以只实现“移除记录”。

## 17.4 拖放移动节点

用户可以通过鼠标拖动资源树中的节点，将其移动到另一节点下。

**实现机制**：

1. 启用 `QTreeWidget` 的 `dragEnabled`、`acceptDrops`、`dropIndicatorShown`。
2. 自定义 `dropEvent`，完全接管拖放逻辑。
3. 根据落点位置（`OnItem` / `AboveItem` / `BelowItem`）解析目标父节点。

**禁止条件**：

- 根节点不可移动。
- 不能将节点移动到自身下。
- 不能将节点移动到自己的子节点下（防止循环引用）。

**移动流程**：

1. 计算新 `folder_path`（基于目标父节点路径 + 节点名称）。
2. 检查磁盘目标位置是否已存在同名文件夹，若存在则拒绝。
3. 若源文件夹存在，计算其总大小。
4. **大小超过 100MB 时弹出确认对话框**。
5. **先移动磁盘文件夹**（`shutil.move`），确保父目录存在。
6. 磁盘移动成功后，调用 `database_service.move_node()` 更新数据库。
7. `move_node()` 会更新 `parent_id`，并递归更新该节点及其所有子节点的 `folder_path`。
8. 刷新资源树。

**磁盘移动失败的处理**：

如果 `shutil.move` 抛出异常（权限不足、路径过长等），直接显示错误弹窗，**不执行数据库更新**，保证数据一致性。

# 20. 错误处理要求

所有错误都应该：

1. 捕获异常。
2. 显示友好提示。

# 21. 性能要求

第一版性能目标：

1. 启动后 2 秒内显示主窗口。
2. 打开项目后不扫描全部图片，只加载逻辑树。
3. 点击节点时再加载该节点下的文件。

6. 缩略图缓存命中时，资源卡片应快速显示。
7. 不要在主线程执行大量图片处理。

实现建议：

1. 使用 `QThreadPool` 或 `QRunnable` 做缩略图任务。
2. 使用懒加载。
3. 使用缓存。
4. UI 刷新时批量更新，避免每生成一个缩略图就全局刷新。

# 43. 已实现功能变更记录

以下记录从原始开发规格到当前工程实际实现的主要变更。

## 43.1 移除的功能

| 功能 | 变更原因 |
|------|---------|
| **节点类型系统**（root/category/character 等） | 用户认为分类无意义，已完全移除。所有节点统一由层级结构管理。 |
| **手动绑定文件夹** | 改为自动计算。每个节点的 `folder_path` 由程序根据父节点路径 + 节点名自动推导，创建节点时自动在磁盘建文件夹。 |
| **Inspector 中的创建时间 / 更新时间显示** | 精简右侧属性栏，移除不常用信息。 |
| **Inspector 中的标签显示** | 精简右侧属性栏。 |
| **"保存备注"按钮** | 改为失去焦点自动保存，减少用户操作步骤。 |
| **FileWatcherService** | 未实现。第一版通过手动刷新处理外部变化。 |
| **asset_card.py 独立文件** | `AssetCard` 内嵌在 `asset_grid_panel.py` 中。 |
| **config.json 项目配置** | 项目级配置统一存储在 SQLite `settings` 表中。 |

## 43.2 新增的功能

| 功能 | 说明 |
|------|------|
| **拖放移动节点** | 鼠标拖动节点到另一节点下，同步移动磁盘文件夹，>100MB 需确认。 |
| **自动打开上次项目** | 启动时通过 `QSettings` 自动恢复上次打开的项目目录。 |
| **最近项目菜单** | 工具栏"项目"下拉菜单记录最近 5 个项目，点击直接打开。 |
| **窗口自动最大化** | `MainWindow.showMaximized()` 启动即全屏。 |
| **删除节点时删除磁盘文件夹** | 删除对话框增加复选框"同时删除磁盘上的对应文件夹"。 |
| **Inspector 删除文件后自动刷新网格** | 删除文件后发射 `file_deleted` 信号，`MainWindow` 连接至 `grid_panel.refresh()`。 |
| **文件大小自动格式化** | 显示为 B / KB / MB / GB，而非原始字节数。 |
| **文件夹大小检查** | `FileSystemService.get_folder_size()` 递归计算文件夹大小。 |
| **数据库 move_node / can_move_node** | 支持节点移动和循环引用检测。 |
| **ensure_folder / rename_folder** | `FileSystemService` 新增文件夹辅助方法。 |

## 43.3 变更的功能

| 功能 | 原设计 | 当前实现 |
|------|--------|---------|
| **新建节点对话框** | 名称 + 类型 + 是否创建文件夹 + 文件夹路径 | 仅保留名称输入 |
| **添加文件** | 若节点有 folder_path 则复制，否则询问 | **总是复制到节点对应文件夹** |
| **顶部工具栏** | "打开项目"按钮 | "项目"下拉菜单（新建 / 打开 / 最近项目） |
| **资源树右键菜单** | "打开绑定文件夹" | "打开文件夹" |
| **属性面板布局** | QFormLayout 平铺所有信息 | QVBoxLayout 分区排版，名称和备注置顶 |
| **文件选择对话框默认过滤器** | Images | All Files |
